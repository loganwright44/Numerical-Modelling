In chapter 8 we used the Ising model to study magnetism and phase transition. Explain what a phase transition is and how you can identify the temperature at which one occurs.

Answer to Question 1¶

From a basic interpretation, a phase transition is the process of a state change for some material such as solid $\rightarrow$ liquid, liquid $\rightarrow$ to gas, etc. From a newer level of understanding from reviewing ideas and concepts from Statistical Mechanics, a phase transition has a lot to say about the orientations of physically characterized parameters such as electron spin. One of the key ideas of phase transition is that the orientation of spin (and hence its magnetic influence) in regions around it influence how the other spin states are also oriented. The Ising Model in conjunction with Monte-Carlo methods allows for a statistically accurate view of how these phenomena can be characterized by temperature changes in the system. And hence, this would attempt to describe the phase transition phenomena by use of statistical mechanics and random methods.

Answer to Question 2¶

When we used methods for modelling large systems such as Molecular Dynamics, it was helpful to incorporate a factor which was a function of temperature which would increase or decrease the velocity influence on the individual particles. This effectively put a buffer on the instability which would have otherwise been introduced by freely flowing particles. In that case, we would have expected that a small instability could lead to the whole system of particles moving together (after a sufficient amount of time) in a very VERY fast flowing 'fluid-like' state. For understanding the melting phase transition, we would expect that at some certain critical melting temperature $T_{\text{crit}}$ that the cluster of interacting 'molecules' would transform from a rigid form (where every particle had a relative region to stay within) to a form of higher flexibiility (where positions of particles are not necessarily bound to any one region). Like I said before, putting some buffer variable (as a function of temperature) as a coefficient to limit the incrementing position parameters would allow for retainment of stability while still maintaining valid characterization of when the phase transition is supposed to occur.

  1. The site energy for this system is similar, though not identical, to what it was for the two-state system. The interaction energy for the Potts model is given by: $$ H_i = -J \sum_j \delta(s_i,s_j)$$ where $\delta(s_i,s_j)$ is the Kronecker delta, which equals one whenever $s_i = s_j$ and zero otherwise. # Ising Model using the algorithm depicted above
import numpy as np
from random import random, randint, choice
import matplotlib.pyplot as plt

class PottsModel:
    def __init__(self, N : int, J : float = 1.0):
        self.N = N
        self.J = J
        self.choices = [- 1.0, 0.0, 1.0]
        self.lattice = np.zeros((N + 2, N + 2))
        self.FillLattice()
        self.kB = 1
        # Parameters for the characterization of the overall lattice
        self.EVariance = 0.0
        self.E = 0.0
        self.M = 0.0
    
    def FillLattice(self):
        """Randomly fills the lattice with 75% spin up

        Returns:
            None
        """
        # Assigns random spin states to the lattice points on the grid
        for i in range(1, self.N + 1):
            for j in range(1, self.N + 1):
                self.lattice[i][j] = choice(self.choices)
        
        return None
    
    def UpdateParameters(self):
        """Simply updates the total magetic moment and energy of the lattice into private variables

        Returns:
            None
        """
        # Reset the Energy and variance to 0
        self.E = 0.0
        self.EVariance = 0.0
        
        # Compute the magnetic moment by adding the spins on the lattice
        self.M = np.sum(self.lattice)
        
        for i in range(1, self.N + 1):
            for j in range(1, self.N + 1):
                # Calculate the energy by entering the ij component into the KroneckerDelta function with its neighbor and sum over the 4 neighbor contributions
                self.E += - self.J * (self.KroneckerDelta(self.lattice[i, j], self.lattice[i - 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j - 1]) \
                    + self.KroneckerDelta(self.lattice[i, j], self.lattice[i + 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j + 1]))
        
        for i in range(1, self.N + 1):
            for j in range(1, self.N + 1):
                # Calculate the energy once again, but take the difference from the mean, and square it. This adds to the variance term.
                self.EVariance += (- self.J * (self.KroneckerDelta(self.lattice[i, j], self.lattice[i - 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j - 1]) \
                    + self.KroneckerDelta(self.lattice[i, j], self.lattice[i + 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j + 1]))) ** 2
        
        # Compute and store the variance by dividing by n (sample size) - 1
        self.EVariance /= (self.N * self.N - 1)
        
        return None
    
    def KroneckerDelta(self, s1, s2):
        return 1 if s1 == s2 else 0
    
    def MetropolisAlgorithm(self, T : float):
        """Randomly picks a point in the lattice to update its value to spin up or down depending on its energy with its nearest neighbors

        Args:
            T (float): temperature under evaluation

        Returns:
            None
        """
        # Choose a random point on the lattice to evaluate
        i, j = randint(1, self.N), randint(1, self.N)
        
        # Calculate the energy of that state with the Kronecker delta to compare the spin states
        E = - self.J * (self.KroneckerDelta(self.lattice[i, j], self.lattice[i - 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j - 1]) \
            + self.KroneckerDelta(self.lattice[i, j], self.lattice[i + 1, j]) + self.KroneckerDelta(self.lattice[i, j], self.lattice[i, j + 1]))
        
        # temporarily make a list element with all options except for the one at the ij index
        options = [o for o in self.choices if o != self.lattice[i, j]]
        
        # Since the energy to flip the state is the negative of its current energy, I simply reverse the directions of inequalities which I stated above
        if E >= 0:
            # randomly choose from the other 2 spin states available if the energy can be lowered
            self.lattice[i, j] = choice(options)
        elif random() < np.exp(E / (self.kB * T)):
            # if the energy was already low but we randomly want to flip the state, we randomly choose from another spin state option
            self.lattice[i, j] = choice(options)
        else:
            # otherwise, and as expected if the energy was already minimized, nothing should change at the ij grid point
            pass
        
        self.UpdateParameters()
        return None
    
    def Animate(self, nIters : int, T : float):
        """Animates the Ising model nIters number of times for temp T

        Args:
            nIters (int): number of times to iterate the model
            T (float): temperature evaluated
        """
        for _ in range(nIters):
            self.MetropolisAlgorithm(T = T)
            plt.imshow(self.lattice[1 : -1, 1 : -1], cmap = 'cool')
            plt.draw()
            plt.pause(0.00001)
            plt.clf()
        
        plt.close()
        return
    
    def ParametersVsTemp(self, TMin : float, TMax : float, nPoints : int, nIters : int):
        """Gets the lists and arrays necessary to plot energy and magnetization vs temperature

        Args:
            TMin (float): min temp test to test
            TMax (float): max temp value to test
            nPoints (int): number of temperature points to test in between max and min
            nIters (int): number of Monte-Carlo time steps (the more, the better for accuracy)

        Returns:
            TempArray, EnergyArray, EnergyVarianceArray, MagnetizationArray: arrays for plotting
        """
        
        # Make the arrays and lists to plot against once computation is ended
        T = np.linspace(TMin, TMax, nPoints)
        E = []
        EVariance = []
        M = []
        
        for t in T:
            # Reset the lattice for a new temperature to test
            self.FillLattice()
            
            # I create little parameters e, eV, and m to sum over the values to approximate an average of energy, variance, and magnetization
            e = 0.0
            eV = 0.0
            m = 0.0
            for _ in range(nIters):
                self.MetropolisAlgorithm(T = t)
                
                # Only start accumulating the average if it has been iterated at least 1000 times
                if _ > 1000:
                    # The energy is saved as the sum of the entire lattice, but we only plot the mean
                    e += self.E / (self.N * self.N)
                    eV += self.EVariance
                    # Likewise with M, we only plot the mean
                    m += self.M / (self.N * self.N)
            
            # After running the algorithm nIters number of times, save the energy, variance, and magnetization averages
            E.append(e / (nIters - 1000))
            EVariance.append(eV / (nIters - 1000))
            M.append(m / (nIters - 1000))
        
        return T, E, EVariance, M

# Instantiate the lattice class
latticeStructure = PottsModel(N = 10, J = 1)

Response to Question 3¶

Above, the plots display the expected characterizations of the Potts Model as I had developed an intuition of how these types of systems behave near their critical temperatures $T_c$. My plots may look a little strange as I have plotted from a very low temperature, however the overall behavior around $T_c$ is as expected. The 'specific heat per spin' plot through differentiation method proved to display that characterizing hump near $T_c \approx 1$. An interesting note from this, however, is that the magnetization can be 0 when all spins are aligned, and as such we have some chaotic behavior for our 'magnetization vs Temp' plot where low temperatures exhibit meta-stable minimized energy states where most (if not all) spin states go to 0. In the 'energy vs temperature' plot, however, my expectation is once again met. The energy of the system on average increases as the temperature increases. This translates to less aligned spin states in the lattice and greater entropy involvement. Additionally, we see that the 'specific heat vs temperature' plot given by the Fluctuation-Dissipation Method shows an exponential and quite steady decrease of specific heat as temperature increases. What's more is that the value $T_c \approx 1$ seems to represent yet another important region on this plot where the slope has significant smoothness to it, as opposed to its left side counter part. Lastly, by making use of another characterization which shows that the maximum value for the specific heat as a function of lattice size given by L, we can see that, although the plots are offset by some constant, both plots follow the same slope of $\ln(L)$. This is expected as stated in problem (8.8) from the Giordano text. To end, it will also be important to mention that through the animation shown when running this in a separate window, the time dependence of the 'relaxation' into the steady state from the initial state proves to slow as time moves forward around $T_c$. This is the final result we would expect from the example in problem (8.9) from the Giordano text.

All of these observations of the Potts Model help to support its validity in that it performs on a lattice structure in a very similar way to the Ising Model, which has proven to be successful in describing such phenomena like ferromagnetism. Introducing a third possibility for spin states allows another degree of freedom to the system, which in turn may lead to further success in describing these 'random' and non-linear interactions within large systems. In reality, spin states are never simply up or down, right or left, etc. there is a continuous path to which direction $L_z$, or $m_s$, may point in a quantum mechanical system. Hence introducing greater degrees of freedom helps to approach this more accurate model of the highly detailed, true universe.
